Detailed Description

template<typename TransformType, typename FromSpace, typename BaseType>
class libchemist::orbital_space::DerivedSpace_< TransformType, FromSpace, BaseType >

Models an orbital space defined in terms of another orbital space.

In practice most orbital spaces are defined as a transformation from another space. For example the MOs are usually defined as a linear combination of the AOs.

Template Parameters
TransformTypeType of the object holding the transformation coefficients from FromSpace to this space.
FromSpaceThe type of the object that the transformation is from. Must be default constructable.
BaseTypeType of the class from which this class is derived.

Creates a new DerivedSpace_ with the specified state.

Of note, this ctor will create an instance which owns the "from space" associated with it. Other spaces can alias this space if desired, but use of this ctor guarantees that the "from space" will not change for the duration of this instance.

Creates a new DerivedSpace_ with the specified state.

This ctor is designed to be used when the "from space" instance has already been created and is in use by another "to space". For exmaple one would use this ctor to construct the virtual MO space using the AO space stored in the already constructed occupied MO space. Note that if the "from space" is modified elsewhere the instance aliased within this class will reflect those changes.

Returns the transformation coefficients to go from from_space to the current space.

Assuming the coefficients are stored in a standard matrix, the convention is that the basis set for the rows is from_space and the current orbital space is the basis set for the columns of the transformation matrix. For more complicated tensors this convention is generalized.
