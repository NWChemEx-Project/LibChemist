Detailed Description

The CTors in this group allow a user to set the state of the Atom via the CTor.

The CTors in this section can be used to create a Molecule initialized with a particular state.

The state of the Atom is set by parsing the list of input arguments by type. The following list first lists the "capture group" (in brackets) followed by how that group will be interpreted.

  • [double, double, double] the input coordinates.
  • [std::array<double, 3>] the input coordinates.
  • [AtomProperty, double] a property and its value
  • [std::string] the name of the Atom.
  • [std::size_t] the atomic number of the Atom.

When providing input to the CTor, the order of the groups is irrelevant.

Parameters
[in]xThe "x" Cartesian coordinate.
[in]yThe "y" Cartesian coordinate.
[in]zThe "z" Cartesian coordinate.
[in]propThe enum corresponding to the AtomProperty to set.
[in]valueThe value for the property
[in]nameThe name of the atom (typically the atomic symbol)
[in]ZThe atomic number of the Atom
[in]argsThe remaining arguments to be parsed
Template Parameters
ArgsThe types of the remaining arguments to be parsed.
Exceptions
std::bad_allocif there is insufficient memory to add a new property. Strong throw guarantee.

The Ctors in this section use look for a particular capture group. The available capture groups are:

  • [Multiplicity] denotes that the wrapped value should be regarded as the multiplicity of the molecule
  • [Charge] denotes that the wrapped value should be utilized as the charge of the molecule
  • [Atom] provides an atom for the molecule

With the exception of Atom instances, a user may not supply more than one of any other instance; attempting to provide multiple charges or multiplicities will result in a compile error. The order of the capture groups is irrelevant.

Parameters
[in]multThe value to set the multiplicity of the molecule to.
[in]cThe value to set the charge of the molecule to.
[in]aOne of the atoms in the molecule.
[in]argsThe remaining unparsed arguments
Template Parameters
ArgsThe types of the remaining unparsed arguments.
Exceptions
