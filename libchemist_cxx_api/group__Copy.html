Detailed Description

CTors and assignment operators for transferring the state of an Atom instance.

Functions for setting the state of the current PeriodicTable instance off of another instance.

Functions for replicating the state of another MoleculeManager instance.

CTors and assignment operators for copying/taking over the state of another AoSFacade instance.

CTors and operators for deep-copying or taking ownership of another AoSElement instance.

Functions for copying and moving an AOShell PIMPL.

Can be used to construct BasisSetMaps with state obtained from another instance.

Can be used to construct BasisSetManagers with state obtained from another instance.

Parameters

Exceptions
std::bad_alloccopy ctor/assignment operators throw if there is insufficient memory to allocate the new state. Strong throw guarantee. All move functions are no throw guarantee.

Given the polymorphic nature of this class we have disabled moving and insisted that copying goes through the clone() function. The copy ctor is protected so that derived classes can use it to implement the clone function.

The biggest gotcha with this class is that the copy CTor is a deep copy. What this means is that deep copying an AoSElement will sever any link it has with the corresponding AoSFacade. This is to prevent const issues from arising.

As a polymorphic object copying/moving incorrectly can lead to problems. We leave it up to the Atom class to implement everything besides deep copy. Deep copy can be done by invoking the clone method, which is most naturally written in terms of the derived class's copy ctor, hence the reason that the copy ctor of this class is protected (i.e. so it can only be used by the derived class) while all others are deleted.
