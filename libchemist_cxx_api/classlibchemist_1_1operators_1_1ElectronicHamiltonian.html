Detailed Description

Hamiltonian focused exclusively on interactions involving electrons.

The ElectronicHamiltonian is at the moment a strong type meant to signify that the Hamiltonian shouldn't contain any terms which do not involve electrons. It's primarily used as an input to correlated methods.

libchemist::operators::ElectronicHamiltonian::ElectronicHamiltonian ( const Hamiltonian &  H)
explicit

Convenience ctor for creating an ElectronicHamiltonian from the full system Hamiltonian.

This function will loop over the terms in H and pull out those which depend on one or more electrons. The resulting ElectronicHamiltonian instance will be initialized with copies of those terms.

Parameters
[in]HThe Hamiltonian we are partitioning.
Exceptions
std::bad_allocif there is a problem allocating memory for the new instance. Strong throw guarantee.
