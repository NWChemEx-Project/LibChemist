Detailed Description

template<typename T>
struct chemist::detail_::ViewTraits< T >

Code factorization for TMP associated with views.

A pattern commonly used throughout Chemist involves a container returning views of its elements instead of the actual elements. Each view is typically a class like View<T> with the idea being that the template class Viewacts like a reference to an object of typeT. IfT` is an unqualified type (notably is not const-qualified), than the view acts like a read/write reference. If T is however a const-qualified type, than the view behaves like a read-only reference. Suffice it to say that properly implementing a const-correct view requires some template meta-programming, which we have factored out into the ViewTraits class.

Template Parameters
TThe type the view is a view of, e.g. for PointView T would be U.

template<typename T >
template<typename U >
using chemist::detail_::ViewTraits< T >::apply_const = std::conditional_t<is_const_v, const U, U>

Makes an un-qualified type U have the same const-ness as T.

This helper will add const to U if T is const-qualified. Otherwise the helper will return U. For example:

using T = Point<float>;
// X will be int
using X = ViewTraits<T>::apply_const<int>;
// Y will be const int
using Y = ViewTraits<const T>::apply_const<int>;

Makes an un-qualified type U into a pointer with the same const-ness as T.

This helper will add const and a pointer to U if T is const-qualified. Otherwise the helper will return a mutable pointer to U. For example:

using T = Point<float>;
// X will be int*
using X = ViewTraits<T>::apply_const_ptr<int>;
// Y will be const int*
using Y = ViewTraits<const T>::apply_const_ptr<int>;
