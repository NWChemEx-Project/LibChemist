Detailed Description

template<typename PointType>
class chemist::PointView2< PointType >

Allows viewing data as if it was a PointType object.

PointView objects act like references to a Point object. More specifically, the state inside a PointView is an alias of data owned by another object (usually a PointSet or related classes).

TODO: This class should replace PointView, at which point we should drop the 2 from the name. This requires switching over Chemist's basis set component.

Template Parameters
PointTypeThe type of Point<T> object this class is behaving like a view of. Assumed to be either Point<T> or const Point<T> (T being either float or double).

template<typename PointType >
PointView2& chemist::PointView2< PointType >::operator= ( point_reference  point)
inlinenoexcept

Sets the value of the aliased Point to point.

Note
This is NOT the copy assignment operator!

This operator does not cause *this to alias point, but instead modifies the values of the Point aliased by *this. i.e. if before calling this method *this has state {px, py, pz} with *px = 1.0, *py = 2.0, and *pz = 3.0 (px, py, and pz being pointers to the x, y, and z coordinates respectively) then after calling this method, px, py, and pz will still point to the same memory locations, but now *px = point.x(), *py = point.y(), and *pz = point.z().

If you want to change the Point this aliases you need to call the copy assignment operator, *i.e. do something like:

Point r0, r1;
PointView me(r0);
r0 = PointView(r1);
Parameters
[in]pointThe value we want to assign to the aliased Point.
Returns
*this after modifying the aliased Point.
