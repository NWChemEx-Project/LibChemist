Atom Retrieval Functions

Returns an Atom instance pre-loaded with the data for the specified atomic number.

There are two masses one can reasonably use for an atom: the abundance- weighted mass and the isotope mass. We provide two different functions to distinguish between these use cases. get_atom returns the abundance-weighted variant, whereas get_isotope returns an Atom whose mass is for a particular isotope.

Parameters

[in]symThe atomic symbol of the atom to create. Symbol is case-insensitive.
[in]mass_numThe mass number of the isotope to create.
Returns
The requested Atom
Exceptions
std::out_of_rangeif Z is not in the range [1, max_Z()), sym is not a recognized atomic symbol, or no isotope data is available. Strong throw guarantee.

Detailed Description

A class to contain Atom instances.

The PeriodicTable class manages Atom instances, which are populated at runtime using the provided insertion methods. The class also serves as a hub for retrieving other atomic data that is not typically stored with an Atom instance.

size_type chemist::PeriodicTable::max_Z ( ) const
noexcept

Returns the maximum atomic number for which an Atom instance can be made.

For the most part retrieving an Atom instance amounts to specifying the mass and atomic symbol for the atom. Atom instances are added at runtime through the insertion methods, so it is not guaranteed that an Atom for each atomic number in the range [1, max_Z()) exists. It should also be noted that some properties may not be available for all atomic numbers in the range [1, max_Z()).

The largest atomic number for which an Atom instance exists
