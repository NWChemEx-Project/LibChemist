Detailed Description

CTors and operators for deep-copying or taking ownership of another AoSElement instance.

CTors and assignment operators for transferring the state of an Atom instance.

CTors and assignment operators for copying/taking over the state of another AoSFacade instance.

The biggest gotcha with this class is that the copy CTor is a deep copy. What this means is that deep copying an AoSElement will sever any link it has with the corresponding AoSFacade. This is to prevent const issues from arising.

Parameters
[in]rhsThe instance to copy/take ownership of. If rhs is moved, after the move it will be in a valid, but otherwise undefined state.
Returns
Assignment operators return the same instance now containing a the state of rhs.
Exceptions
std::bad_allocif there is insufficient memory to copy rhs. Strong throw guarantee. Move Ctor/assignment are no throw guarantee.

Copies of AoSFacade instances are deep copies.
